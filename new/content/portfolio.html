Storm8 Confluence Pages

The last few years at Storm8 I focused on tools, libraries and documentation, and promoted general knowledge sharing. Below is a sample of Wiki pages I created during that time.

General Engineering

  • Code Review Checklist

    A code review checklist I created for engineering. Another engineer added the server side items.

  • Unity GameObject Profiling

    A tool I created to help monitor how many coroutines, GameObjects and Update methods are in the current scene. Games are all about memory usage and responsiveness.

  • Visual Debugging Tools

    Tools I created to help debug our games. It helps QA, dev, and anyone during play-tests to find component names of anything you see on screen.

  • Structs vs Classes

    A short blog article for engineering about structs (value types) vs classes and when to use them.

Design Documentation

  • Foundation Library Design

    A library meant to be shared between micro-services. It's based on git subtree. I started the implementation for the client, and another engineer started on back-end code.

  • WorkerQueue Design

    A set of classes that allow one coroutine to process many workers, instead of starting many coroutines. Similar to a job scheduler that supports additions and cancelations of tasks.

  • PowerManager Design

    A design of a configurable power manager which allows mobile devices to run at 30 FPS when there is no interaction, or when no animations are running.
