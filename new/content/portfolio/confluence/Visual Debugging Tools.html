Visual Debugging Tools

Skip to end of metadata
  • Created by Mike Austin, last modified by Sukwon Lee on Mar 16, 2016
Go to start of metadata

Here you'll find a few iOS tools to help when you're in a play-test, trying to verify dialog layouts, or simply when you want to add gems or change levels.

 

Table of Contents

  • 1C_ENABLE_LOCALIZATION_ASSERTS
  • 2Long-press Xib Information Debug
  • 3Long-press Embedded testFrame
  • 4Long-press Dialog Preview Debug

 

C_ENABLE_LOCALIZATION_ASSERTS

In AppInit.mm or AppDelegate.mm setupEnvironment:qaHostOption:, depending on game type, set

[ConfigManager instance].C_ENABLE_LOCALIZATION_ASSERTS = YES;

to enable showing asserts if a localization key is missing.

 

Long-press Xib Information Debug

Press and hold anywhere on the screen to find the deepest Xib view and it's name.  This is useful for artists who are skinning Xibs, for giving more details in bug reports, and generally finding the name of Xibs.

Dialogs and view controllers automatically show their named when opened.  If not working, ask an engineering lead to check XibDebugTouchGestureRecognizer.

 

Long-press Embedded testFrame

Press and hold the Storm8 button to open an embedded testFrame.  This is useful for changing your levels, giving yourself gems, and changing experiment parameters.

Tap 'Update', and in a few seconds the game will log in again with the updated values.  Merge PromotionButtonView.mm into your branch to enable.

 

After giving yourself some levels, you may encounter a number of popups. By clicking "disable all popups" button at the bottom right, all popups will be disabled temporarily. 

Long-press Dialog Preview Debug

Press and hold the Storm8 Menu close button to open dialog debugging.  This is useful to show any dialog in game, without having to navigate to it or force it in code.

Any method starting with 'initWithDummyData' will be listed in the second section.  Merge PromotionMenuView.mm into your branch to enable.

 

Example:

 

@implementation LevelUpView
- (instancetype)initWithDummyDataForLevel10 {
    if (self = [super init]) {
        [self setupWithLevel:10];
    }
 
    return self;
}

@end
