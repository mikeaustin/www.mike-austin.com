DatabasePlus 2.0: What's New
A user's guide for adjusting to the new and improved product

Summary
The most noticeable difference you'll find with DatabasePlus 2.0 is that we've redesigned the interface. Instead of a vertical toolbar with drawers, there is now a horizontal toolbar with popup menus for rapidly moving between elements when designing a database. Also, when using a database, you'll find the vertical toolbar is replaced with explicit links within the body of the record browser. The navigation and structure of the page previously called "My Bitlockers" is greatly improved. Because this page is where you'll find links to all your various databases, reports and workgroups, we've renamed it "My Bitlocker Desktop".

There's also a fundamental change to the structure of the product. You'll find the meaning of some terms has changed. With the addition of relational, there are now multiple "tables" within a database, each of which contains fields and value lists. What had been called "Views" before, where layout information was stored, has now become a relational view where fields can be combined from multiple tables into a single "virtual table". The formatting information has now been pushed down to reports.

For more detail on these changes and more, please browse the index below.

1. Navigation & Interface
Instead of a vertical toolbar with drawers, there is now a horizontal toolbar with popup menus for rapidly moving between elements when designing a database. The first popup menu allows you to select the task you wish to perform. Choose between browse and author. The second popup menu allows you to specify which object type you want to browse or author. Choose between database, tables, views and reports. The third popup menu only shows up when you've selected an object other than database and allows you to select exactly which table, view or report which you wish to browse or author.

Whereas in 1.8 almost all functionality was located in the left hand toolbar drawers, in 2.0 you'll find a more intuitive, inline approach to the interface. Simply follow the instructions which accompany each of the menus and wizards.

Index


2. Database Structure & Terminology
With the addition of relational functionality, there are now multiple tables within a database, each of which contains fields and value lists.

What had been called views before, where layout information was stored, has now become a relational view where fields can be combined from multiple tables into a single "virtual table". The formatting information has now been pushed down to reports.

Reports specify a base table or view, but add all the formatting information that had been stored with v1.8 views.

In Reports, "multiple view" support has been replaced with "tabs". Tabs are groups of fields. They have names and order, so forms can be constructed with a summary area and a collection of tabs containing additional information.

Guide to Terminology

  In 1.8 it was... Now in 2.0 it is...
database A single table of records Multiple tables of records, fields have relational functionality across tables
table A type of view Independent groups of records that can relate to each other
view Layout information Relational virtual table, layout information now in reports
report Some layout information, multiple views All layout information, tabs

NOTE: Calculated fields can now only be created in views (not tables.) Views only contain derived information from other tables. Tables only contain actual values that are entered or imported by the user.

3. Security Enhancements
Access control lists replace the use of task based security (publish, survey, etc.) for finer control of access privileges. In 1.8, task based security settings were found in the settings dialog. Now access control lists are at the table level.

Workgroups are now accessible from the desktop. Groups of users can be added to access lists in the form of workgroups. To add users to workgroups, you now "invite" them via email rather than looking them up in the system.

Once a user has accepted an invitation to any workgroup, they become members of the account and can then be quickly added to any workgroup, or directly to any access control list.

NOTE: To add a user directly to an access control list, they must first be invited to a workgroup and accept the invitation.

Index


4. Publishing Enhancements
There is now a wizard for creating links that can be embedded in other sites.

There is an option in the wizard to indicate if the embedded element is a report or survey. This replaces the use of the security setting to determine if it should be a survey.

Index


5. New & Improved Desktop
Click the "Create a New Database" link at the top of your desktop to get started. Click the "Author" link to the far right of any database to setup and structure your database.

Workgroups are now at the desktop level. This gives you the added advantage of being able to give one group of users access to more than one database without having to recreate the group.
