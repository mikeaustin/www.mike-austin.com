Is there a more idiomatic way to do the following?
var = hash[key].nil? ? nil : hash[key].downcase Note that if hash[key] is nil, I want nil assigned to var, so this won't work:
var = hash[key].downcase unless hash[key].nil?
Obviously I could do this, but I'm trying to keep it on one line:
var = hash[key]
var = var.downcase unless var.nil?
Kirk Haines
I usually do:
hash = { :key1 => nil , :key2 => "aBcD" }
var = hash[:key1] && hash[:key1].downcase
var # => nil
var = hash[:key2] && hash[:key2].downcase
Charles Calvert
Using the short-circuit evaluation of && to avoid calling downcase. Interesting.
Kirk Haines
Notice the nuance that if hash[:key1] is false, that will be assigned to var.
